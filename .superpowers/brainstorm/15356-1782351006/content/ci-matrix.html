CI 矩阵改造

GitHub Actions 全架构覆盖

当前 CI → 目标 CI

构建矩阵对比
❌ 当前
strategy:
  matrix:
    os: [ubuntu, windows, macos]
    # 无架构区分
    # E2E 仅 ubuntu
    # 无 ARM64 runner
✅ 目标
strategy:
  matrix:
    include:
      - os: macos-14
        arch: arm64
      - os: macos-13
        arch: x64
      - os: windows-latest
        arch: x64
      - os: windows-11-arm
        arch: arm64
      - os: ubuntu-latest
        arch: x64
      - os: ubuntu-24.04-arm
        arch: arm64

CI Job 结构

3 个 Job：lint → build → e2e
┌─────────────────────────────────────────────────┐
│  lint (ubuntu-latest, 单次)                      │
│  ├── ESLint                                      │
│  └── Type check                                  │
├─────────────────────────────────────────────────┤
│  build (6 节点矩阵)                              │
│  ├── macos-14 arm64  → DMG arm64                │
│  ├── macos-13 x64    → DMG x64                  │
│  ├── windows-latest x64  → NSIS x64 + MSIX x64 │
│  ├── windows-11-arm arm64 → NSIS arm64 + MSIX   │
│  ├── ubuntu-latest x64  → AppImage x64          │
│  └── ubuntu-24.04-arm arm64 → AppImage arm64    │
├─────────────────────────────────────────────────┤
│  e2e (3 节点，仅 x64)                            │
│  ├── ubuntu-latest x64 (Xvfb)                   │
│  ├── windows-latest x64                         │
│  └── macos-13 x64                               │
│                                                   │
│  ⚠️ ARM64 E2E 跳过：Playwright 在 ARM64 CI 中    │
│     稳定性不足，仅做构建验证                       │
└─────────────────────────────────────────────────┘

关键配置细节

macOS ARM64 runner：使用 macos-14（M1 runner），macos-13 为 Intel runner，分别构建对应架构。

Windows ARM64 runner：使用 windows-11-arm（GitHub 2025 新增的 ARM64 runner）。

Linux ARM64 runner：使用 ubuntu-24.04-arm（GitHub ARM64 runner）。

E2E 仅 x64：ARM64 CI 环境下 Playwright 浏览器下载和渲染不稳定，暂时跳过，仅在构建阶段验证编译通过。

构建产物上传：每个矩阵节点上传对应架构的安装包到 GitHub Release / Artifacts，便于分发。

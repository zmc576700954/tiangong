2. MSIX 专属配置

"msix": {
  "identityName": "com.bizgraph.app",
  "publisher": "CN=BizGraph",
  "publisherDisplayName": "BizGraph",
  "languages": ["zh-CN", "en-US"],
  "store": false
}

⚠️ store: false 表示先做侧载分发，后续上架 Microsoft Store 时再改为 true 并补充签名配置

3. 构建脚本扩展

package.json scripts
"build:win":       "electron-builder --win"              // NSIS x64+arm64
"build:win:msix":  "electron-builder --win msix"          // MSIX x64+arm64
"build:mac":       "electron-builder --mac"               // DMG arm64+x64
"build:linux":     "electron-builder --linux"             // AppImage x64+arm64
"build:all":       "electron-builder --mac --win --linux" // 全平台

4. Windows ARM64 libsql 兼容策略

原理：Windows 11 ARM64 内置 x64 模拟层（WoA64），x64 Node.js 原生模块可透明运行。

实现：electron-builder 打包时，Windows arm64 构建仍包含 @libsql/win32-x64-msvc 原生包。运行时通过 x64 模拟层加载，无需代码改动。

兜底：在 src/main/database.ts 初始化时检测架构不匹配，若 x64 模拟层不可用则 fallback 到 libsql HTTP 远程模式（连接 Turso 云端），并提示用户。

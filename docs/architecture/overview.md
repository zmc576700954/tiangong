## 核心模块

### 1. Agent 适配器层

技术核心。每个 Agent CLI 有不同的交互协议，BizGraph 提供统一抽象：

- `BaseAdapter`: 抽象基类，定义扩展接口
- `ClaudeCodeAdapter`: Claude Code 适配器（stdin/stdout 通信）
- `CodexAdapter`: Codex CLI 适配器
- `OpenCodeAdapter`: OpenCode 适配器

### 2. 范围守卫 (ScopeGuard)

系统层面的文件变更保护：

- 执行前备份白名单文件
- 执行中监控文件系统（chokidar）
- 越界写入时自动终止 Agent 并回滚

### 3. 双图模型

- **真实图 (Production)**: 产品经理维护的业务蓝图，节点状态从"草稿"到"已发布"
- **开发图 (Development)**: 从占位节点派生的实现副本，开发者在此工作

### 4. 节点状态机

```
draft → confirmed → developing → testing → review → published
```

占位节点是特殊的 "placeholder" 状态，用于在真实图上标记"待实现"的区域。

## 数据流

1. 用户在画布上操作（创建/编辑/连接节点）
2. 渲染进程通过 IPC 调用主进程 API
3. 主进程操作 LibSQL 数据库
4. 数据变更通过 Zustand 状态管理同步到 UI
5. 启动 Agent 时，主进程生成范围上下文并启动子进程
6. Agent 输出通过 IPC 事件流推送到渲染进程
